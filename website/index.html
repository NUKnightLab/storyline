Introducing Storyline

Storyline is an open-source tool that enables anyone to build an annotated, interactive line chart. Note that this tool is in beta. That means it’s a work in progress. We’re floating our unfinished work to get feedback and see if people find it useful before we build the easy-to-use authoring tools that make this a better fit for beginners.

To make Storyline as flexible as possible, we've just included the chart, axis labels, and cards. Headlines, context, more specifics on the data sources and credits--we expect you'll include what you want outside of Storyline. We built the chart up top using Real Median Household Income data from FRED.

Current limitations

  1. We’re focused on line charts for now. The x-axis will accept only time/date formats.
  2. Our mobile-first design requires you to keep it short. No more than 12 cards per story, titles limited to 54 characters, and paragraphs limited to 200 characters.
  3. Storyline works best with fewer than 800 data points.

Try the Beta

For this beta version, we're doing all of our documentation on Github. Check out the README for a walk-through.

Note that this working version requires you to run a local web server. There are many ways to do this. Here's a guide to running a local webserver on MacOS. Windows users may might try installing WAMP. If these options seem daunting, rest assured that this is temporary: as we learn from the beta and make refinements to StorylineJS, we'll also develop easier ways for everyone to use it.

Find a bug? If you are confident you have found a bug, please report it as a GitHub issue. Be sure to include detailed instructions on how to reproduce the bug.

Help

Thanks so much for building a Storyline. It’s a work in progress, and we’d love to know what’s working and what’s not. We’re inviting people to submit issues on Github to let us know what could be better. Make sure to check out our existing plans before submitting.

Want us to build out the authoring tool to make this more robust? Have other big ideas that don't feel like they belong on Github? Feel free to send us feedback.
